# 前端改進實施指南 (2.1 Session Token 改進)

**優先級**：🟠 中優先 | **預計完成**：2026年4月底

---

## 1. 改進概要

### 當前狀態
- ❌ Token 存儲在 localStorage
- ❌ Token 在 URL 參數中傳遞
- ✅ 已移除 localStorage.isAdmin

### 目標狀態（方案 A - HttpOnly Cookie）
- ✅ Token 存儲在 HttpOnly Cookie（由後端設置）
- ✅ 不在 URL 中傳遞 token
- ✅ 瀏覽器自動管理 cookie

### 目標狀態（方案 B - Authorization Header）
- ✅ Token 不在 URL 中傳遞
- ⚠️ 仍需在 localStorage 中存儲（如無法使用 Cookie）
- ✅ 使用 HTTP 授權標頭傳遞

## 3. 方案 A：改用 HttpOnly Cookie（推薦）

### 3.1 修改 callApifetch 函數

**文件**：`js/core.js` 第 143-194 行

```javascript
/**
 * 透過 fetch API 呼叫後端 API。
 * 改進版本：使用 HttpOnly Cookie（後端需配合）
 * @param {object} params - 包含 action 和所有其他參數的物件
 * @param {string} [loadingId="loading"] - 顯示 loading 狀態的 DOM 元素 ID
 * @returns {Promise<object>} - 回傳一個包含 API 回應資料的 Promise
 */
async function callApifetch(params, loadingId = "loading") {
    // ✅ 改進 1：不再讀取 localStorage 中的 token
    // ❌ 已移除：const token = localStorage.getItem("sessionToken");
    
    // 1. 構造 URLSearchParams 物件
    const searchParams = new URLSearchParams(params);
    
    // 2. 移除 token URL 參數（改由 Cookie 自動提供）
    // ❌ 已移除：searchParams.set("token", token);
    
    // 3. 加入 callback 參數以使用 JSONP 避免 CORS 問題
    const callback = 'callback' + Date.now() + Math.random().toString(36).substr(2, 9);
    searchParams.set("callback", callback);
    
    // 4. 構造最終 URL
    const url = `${API_CONFIG.apiUrl}?${searchParams.toString()}`;
    
    // 顯示指定的 loading 元素
    const loadingEl = document.getElementById(loadingId);
    if (loadingEl) loadingEl.style.display = "block";
    
    try {
        // ✅ 改進 2：添加 credentials 選項以包含 Cookie
        const response = await fetch(url, {
            method: 'GET',
            mode: 'cors',
            credentials: 'include',  // 🌟 關鍵改動：包含跨域 Cookie
            headers: {
                'Content-Type': 'application/x-www-form-urlencoded'
            }
        });
        
        // 檢查 HTTP 狀態碼
        if (!response.ok) {
            throw new Error(\`HTTP 錯誤: \${response.status}\`);
        }
        
        // 解析 JSON 回應
        let text = await response.text();
        try {
            return JSON.parse(text);
        } catch (jsonError) {
            // 若回傳類似 callback({...}) 的 JSONP，嘗試抽取 JSON
            const match = text.match(/^[^(]*\\((.*)\)\\s*;?\\s*\$/s);
            if (match && match[1]) {
                return JSON.parse(match[1]);
            }
            throw jsonError;
        }
    } catch (error) {
        // 處理網路或其他錯誤
        showNotification(t("CONNECTION_FAILED"), "error");
        console.error("API 呼叫失敗:", error);
        throw error;
    } finally {
        // 不論成功或失敗，都隱藏 loading 元素
        if (loadingEl) loadingEl.style.display = "none";
    }
}
```

### 3.2 修改登入流程

**文件**：`js/app.js` 第 274-318 行

```javascript
// 改進後的登入處理邏輯

// 處理 otoken 換取 sessionToken 的流程

try {
    console.log(currentLang);
    const res = await callApifetch({ 
        action: 'getProfile', 
        otoken: otoken, 
        languag: currentLang 
    });
    
    if (res.ok) {
        // ✅ 改進 1：不再存儲 token 到 localStorage
        // ❌ 已移除：localStorage.setItem("sessionToken", res.sToken);
        
        // ✅ 後端在 Set-Cookie 中設置 sessionToken
        // 瀏覽器會自動保存 Cookie
        
        // 清除 URL 中的 otoken
```

## 4. 方案 B：改用 Authorization Header

如果後端無法實現 HttpOnly Cookie，可改用 Authorization Header：

### 4.1 修改 callApifetch 函數

```javascript
async function callApifetch(params, loadingId = "loading") {
    const token = localStorage.getItem("sessionToken");  // 仍需保存
    
    const searchParams = new URLSearchParams(params);
    
    // ✅ 改進：不在 URL 中傳遞 token，改用 Authorization 頭
    // ❌ 已移除：searchParams.set("token", token);
    
    const callback = 'callback' + Date.now() + Math.random().toString(36).substr(2, 9);
    searchParams.set("callback", callback);
    
    const url = `${API_CONFIG.apiUrl}?${searchParams.toString()}`;
    
    const loadingEl = document.getElementById(loadingId);
    if (loadingEl) loadingEl.style.display = "block";
    
    try {
        const response = await fetch(url, {
            method: 'GET',
            mode: 'cors',
            // ❌ 移除：credentials: 'include'（改用 Authorization 頭）
            headers: {
                'Content-Type': 'application/x-www-form-urlencoded',
                'Authorization': token ? \`Bearer \${token}\` : ''  // 🌟 添加授權頭
            }
        });
        
        // ... 其他邏輯保持不變
    }
    
    // ...
}
```

## 5. 檢查清單
