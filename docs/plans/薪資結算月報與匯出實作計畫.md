> 建立：2026-04-26

## 目標

把 Phase L 計算出的「即時估算」固化為**正式月度結算**，讓管理員：

1. 一鍵結算當月所有員工薪資 → 寫入 `payroll/{YYYY-MM}/{userId}`
2. 月報 dashboard 列出全公司結算狀態（未結 / 已結 / 已發）
3. 匯出 Excel 三種格式：薪資清冊、個人薪資單、銀行匯款檔
4. 員工自助查詢已發放的薪資紀錄
5. （後續）LINE 推播月薪結算完成 + 連結到自助頁

**核心理念**：結算 = **快照 + 鎖定**。一旦結算寫入 Firestore，後續即使 dailyStatus / 薪資設定改變，舊月份結算結果不變（除非管理員「重新計算」並覆寫）。

## 名詞與資料模型

### 薪資結算 record（`payroll/{YYYY-MM}/{userId}`）

```js
{
  // ===== 基本識別 =====
  userId: 'U1234',
  name: '林杰',
  monthKey: '2026-04',

  // ===== 結算狀態 =====
  status: 'settled' | 'paid' | 'recalculated',  // settled=已結算未發、paid=已發、recalculated=重新計算過
  settledAt: <Timestamp>,
  settledBy: <admin userId>,
  paidAt: <Timestamp | null>,
  paidBy: <admin userId | null>,

  // ===== 薪資設定快照（避免之後改動影響歷史紀錄）=====
  salaryType: 'monthly',
  monthlySalary: 36000,
  hourlyRate: 150,
  laborInsuranceGrade: 6,
  insuredSalary: 36300,           // 該等級對應月投保薪資
  hasLaborPension: true,
  laborPensionRate: 3,

  // ===== 工時統計快照（aggregate 結果）=====
  laborStats: {
    normal: 152, ot1: 8, ot2: 2,
    rest_ot1: 4, rest_ot2: 6, rest_ot3: 0,
    public_base: 8, public_ot1: 0, public_ot2: 0,
    regular_base: 0, regular_comp: 0, regular_ot: 0,
    equivalentHours: 188.5,
    gross: 175.5, net: 165.5,
  },

  // ===== 應發明細 =====
  pay: {
    normal: 22800,                // 152h × 150
    plainOt: 1740,                // (8 × 4/3 + 2 × 5/3) × 150
    restOt: 2900,                 // (4 × 4/3 + 6 × 5/3) × 150
    publicGuaranteed: 1200,       // 8 × 150
    publicOt: 0,
    regularBase: 0, regularComp: 0, regularOt: 0,
    bonus: 0,                     // 額外津貼/獎金（手動加）
    deductionEarn: 0,             // 額外加項（如全勤獎）
    grossTotal: 28640,
  },

  // ===== 扣繳明細 =====
  deduction: {
    laborInsurance: 871,           // 36300 × 2.4%
    healthInsurance: 563,          // 36300 × 1.55%
    laborPension: 1089,            // 36300 × 3%
    incomeWithholding: 0,          // 所得稅扣繳
    healthSupplement: 0,           // 二代健保補充保費（暫保留）
    other: 0,                      // 其他（手動扣項，如借支）
    deductionTotal: 2523,
  },

  // ===== 實領 =====
  netPay: 26117,                   // grossTotal - deductionTotal

  // ===== 銀行匯款資料 =====
  bankAccount: '玉山銀行 808-1234567890123',  // 從 employees/{userId} 帶
  paymentMethod: 'transfer' | 'cash',

  // ===== 備註 =====
  remark: '',                      // 管理員手動加註
  history: [                       // 重算紀錄
    { at: <Timestamp>, by: <userId>, reason: '修正午休扣減錯誤' },
  ],
}
```

## 規則對齊（採用標準）

### 採用「等價時數 × 時薪」公式（Phase L7 沿用）

**爭議點**：勞基法施行細則第 31 條規定月薪 ÷ 30 ÷ 8 = 平日時薪，
但月薪制本身代表「不論是否上班 30 天都給」，加班費才是額外。
本系統採折衷：

- 月薪制：基本月薪固定 + 加班費按各段倍率算
- 時薪制：等價時數 × 時薪

**計算公式**：

| 項目 | 月薪制 | 時薪制 |
|------|--------|--------|
| 基本工資 | `monthlySalary` (固定) | `normal × hourlyRate` |
| 平日加班 | `(ot1 × 4/3 + ot2 × 5/3) × hourlyRate` | 同 |
| 休息日工資 | `(rest_ot1 × 4/3 + rest_ot2 × 5/3 + rest_ot3 × 8/3) × hourlyRate` | 同 |
| 國定假日基本 | `8 × hourlyRate`（出勤即給） | 同 |
| 國定假日加班 | `(public_ot1 × 4/3 + public_ot2 × 5/3) × hourlyRate` | 同 |
| 例假日工資 | `8 × hourlyRate`（出勤即給） | 同 |
| 例假日補休折現 | `8 × hourlyRate` | 同 |
| 例假日加倍 | `regular_ot × hourlyRate`（規則內已 ×2） | 同 |

**所得稅扣繳**：暫不實作（依員工填報計算複雜，先保留 0），可手動 override。

**二代健保補充保費**：暫不實作（單筆獎金 ≥ 4倍投保薪資 才扣繳，
本系統一般月薪不會觸及，特殊獎金時手動加）。

### Phase M1：結算引擎後端（中，~2 小時）

#### M1.1 Cloud Function `settleMonthlyPayroll`

```js
// admin only, region asia-southeast1
{
  action: 'settleMonthlyPayroll',
  monthKey: '2026-04',
  userIds: ['U1', 'U2', ...] | 'all',  // 'all' 結算全公司
  recalculate: false,                    // true 才允許覆寫已結算
  remark: '',
}
```

行為：
1. 讀取 `payrollBatches/{monthKey}`，若 `isLocked` 拒絕
2. 對每個 userId：
   - 讀 `employees/{userId}` 拿薪資設定
   - 讀 `getAttendanceDetails(monthKey, userId)` 拿 dailyStatus
   - 在後端 enrich（複製 labor-hours 計算邏輯到 functions/src/_payroll.js）
   - 計算 pay / deduction / netPay
   - 寫入 `payroll/{monthKey}/{userId}`，含 history 軌跡
3. 更新 `payrollBatches/{monthKey}` 統計
4. 回傳 { ok, results: [{userId, status, netPay}, ...] }

驗證：
- 月薪制 < 28590 → ERR_BELOW_MIN_WAGE
- 等級不在 1-23 → ERR_INVALID_GRADE
- 已 isLocked → ERR_BATCH_LOCKED

#### M1.2 後端共用 `_payroll.js`

把 `js/labor-hours.js` 的核心算法移植到 `functions/src/_payroll.js`，
避免前後端規則不同步。或共用 npm package（package.json 加 `"./labor"`）。
**初期採複製**，定期手動同步；若有差異測試會抓出來。

#### M1.3 `getPayrollList` / `getPayrollDetail`

- `getPayrollList(monthKey)`：admin 取整月清單（無詳細工時，僅摘要）
- `getPayrollDetail(monthKey, userId)`：admin 或本人，取單筆完整明細
- `getMyPayrollHistory(years?)`：員工自助查歷史 12 筆

## 風險與注意事項

1. **前後端規則同步**：M1.2 採「複製 + 手動同步」，建議加單元測試
   `__tests__/payroll-consistency.test.js` 對相同 input 跑前後端，比對結果一致
2. **月底結算時間點**：員工可能還在補打卡 / 申請補休 → 建議「鎖定截止日 + 預告」
   - 例：每月 5 日結算上月，前 3 日預告，4 日鎖定打卡，5 日結算
3. **重新計算**：必須留 history 軌跡且需理由，避免管理員任意改數字
4. **薪資資料隱私**：
   - 員工自助頁只能看自己（後端 verify userId）
   - 整月清冊 Excel 下載需 admin 權限
   - 銀行匯款檔含敏感帳號，下載前再次確認
5. **負薪資**：扣繳 > 應發（極端情況：月初離職又有借支扣回）→ UI 紅字、需手動 override
6. **跨月加班**：跨日打卡（4/30 22:00 上班、5/1 02:00 下班）目前 dailyStatus 算 4/30 一整段，需確認規則
7. **基本工資年度更新**：每年元旦前手動更新 `MIN_MONTHLY_WAGE_2026` 常數
   - 建議改用 `firestore: settings/laborRates/{year}` 動態讀取
8. **勞保等級分級表年度更新**：同上，現在硬編 2026 表
9. **時薪制特休折現**：時薪制員工特休不休是否折現？需與 user 確認
10. **零工時月份**：如員工整月請假/育嬰假 → 應發 0、扣繳照算（投保薪資依舊扣）

## 驗收標準（Phase M1~M5）

### 結算正確性
- [ ] 月薪 36,000 + 平日 152h + 平日加班 OT1 8h + OT2 2h
      → 應發 = 36,000 + (8×4/3 + 2×5/3)×150 = 36,000 + 1,600 + 500 = **38,100** ✓
      → 扣繳 = 871 + 563 + 1,089 = 2,523 ✓
      → 實領 = **35,577**
- [ ] 時薪 200 + 等價時數 188.5 → 應發 = 37,700
- [ ] 結算後修改 dailyStatus → payroll 紀錄不變
- [ ] 「重新計算」覆寫且寫入 history

### 月報 UI
- [ ] 點員工 [明細] 顯示完整 pay/deduction
- [ ] 合計列正確
- [ ] 5 語言切換無漏字

### Excel 匯出
- [ ] 薪資清冊 Excel：每員工一行 + 合計列、欄位完整
- [ ] 個人薪資單 Excel（多 sheet）：簽收欄留白
- [ ] 銀行匯款檔：金額正確、戶名/帳號完整

### 員工自助
- [ ] 只看到自己的紀錄

- [ ] 下載個人薪資單成功

### 通知

- **儀表板統計**：年度總人事成本、加班費佔比、各部門成本分布
- **稅務支援**：所得稅扣繳查表、二代健保
- **跨幣別**（如外籍員工部分薪資匯外幣）

這些屬於「人資系統」雛形，留待產品成熟後評估。
