# 員工管理介面重設計計畫

> 建立：2026-04-26
> 狀態：規劃中
> 設計參考：Crextio dashboard（行動端 first 響應式 + 多卡片儀表板）

---

## 目標

重新設計 **管理員 → 員工管理 sub-tab**，達成：

1. **單一員工選擇器**（取代目前 2 個重複的 select）
2. **行動端優先的響應式儀表板**（mobile single-column → tablet 2-col → desktop 3-col）
3. **資訊整合**：把目前散落各處的功能集中到一個員工 dashboard
4. **新增分析卡片**：請假統計、連續上工天數、申請歷史等

## 卡片清單

### 1. 員工 Hero 卡片
- 大頭照（LINE picture）
- 姓名（大字）
- 部門（dept badge）
- userId 後 8 碼（小字）
- 加入日期、最後打卡時間

**來源**：`getEmployeeList` 回傳項目

### 2. KPI 統計列（4 格）
| KPI | 算法 |
|---|---|
| 本月工時 | sum(dailyStatus.hours) |
| 本月正常 | sum(min(hours, 8)) |
| 本月加班 | sum(max(hours-8, 0)) |
| 本月請假天數 | count(reason in [LEAVE_APPROVED, VACATION_APPROVED]) |

### 3. 月曆 + 週工時圖
**沿用既有元件**（`renderCalendarWithData` + `renderWeeklyChart`），只是放進新 layout

### 4. 連續上工天數卡片
- 大數字 + 火焰 icon
- 算法：從今天往回算，連續有 PUNCH_NORMAL 的日數
- 中斷規則：缺打卡、請假、休假、國定假日不計入但也不中斷（可選）

### 5. 請假/休假統計卡片
- 列表：年假 X 天、特休 Y 天、病假 Z 天、事假 ...
- 算法：scan 該員工 attendance with audit='v' AND adjustmentType='系統請假記錄'

### 6. 申請紀錄卡片
- 3 個 tab：待審核 / 已批准 / 已拒絕
- 每筆：日期、類型（補打卡/請假/休假）、目標時間、申請時間、狀態
- 待審核項目可直接從這裡 approve/reject
- **後端需擴充** `getReviewRequest` 接受 `userId` + `audit` filter（或新增 `getEmployeeRequestHistory`）

### 7. 打卡紀錄表格
- 表頭：日期、上班、下班、工時、地點、狀態
- 預設顯示本月，可切換月份
- 響應式：手機改 card 模式（每筆一張小卡）
- 來源：`getAttendanceDetails().records.dailyStatus`

### 8. 員工設定區（摺疊）
- 預設摺疊（accordion），減少 first paint 噪音
- 內容：權限/帳號控制 + 打卡政策（沿用既有）

### 9. 動作按鈕列
- 匯出 Excel
- 測試通知
- 下載完整資料（複用 getCompleteAttendanceRecords）

## 後端需求清單

### 修改

**`getReviewRequest`** — 加參數
```js
// 新支援:
{ action: 'getReviewRequest',
  userId?: string,    // 過濾指定員工，未傳=全公司
  audit?: '?'|'v'|'x'|'all',  // 預設 '?' 維持向後相容
  limit?: number      // 預設 200
}
```

回傳 item 加 `audit` 欄位（前端 tab 分群用）。

### 新增（可能）

**`getEmployeeStats`**（如果想把計算搬到後端）
```js
{ action: 'getEmployeeStats', userId, month }
→ {
  totalHours, normalHours, overtimeHours, restHours,
  leaveDays, vacationDays, sickDays, personalDays,
  consecutiveWorkdays, lastPunchAt
}
```

**目前先放前端計算，省一個 endpoint。後端版本之後再做。**

## 風險與注意事項

1. **Tailwind JIT 風險**：複雜 grid class 可能未 build。沿用 inline CSS grid 或 .compiled.css 的既有 utility（grid grid-cols-1 grid-cols-2 grid-cols-3 grid-cols-4 已 build）
2. **資料量**：員工累積打卡可能上千筆，表格需分頁或限制本月
3. **權限**：所有新 API 必須 verifyAdmin
4. **快取**：員工切換時要 invalidate 既有 monthDetail / abnormal cache
5. **漸進式**：每個 Phase 一個 commit，方便回滾
