## 目標

把目前管理員 dashboard 的「工時」單一數字（`day.hours`）細分為勞基法
規則下的多段時數，讓管理員一眼看到：

1. 每日：**正常 / 平日加班 1.34× / 平日加班 1.67× / 休息日 / 例假日 / 國定假日**
2. 全月 KPI：**正常時數、平日加班時數、休息日時數、國定假日時數、例假日時數**
3. 週圖：模式增加「平日加班 / 假日工時」
4. 表格：工時欄展開顯示分段
5. （未來）薪資估算：依時薪 × 倍率計算月薪

**完全不動後端** — 規則用既有 dailyStatus + holidays-client + breakTimes
在前端計算。後端薪資結算（月底報表）暫不在範圍。

## 規則對齊（摘自 docs/rules/薪資與加班計算規則整理.md）

### 日期類型分類（4 種）

| 類型 | 判斷 | 計算規則 |
|---|---|---|
| **平日** | 週一-五 + 非國定假日 + 非補班補休 | 8 小時內正常 / 加班分段 |
| **休息日** | 週六（無國定假日覆蓋） | 工時全為加班，3 段倍率 |
| **例假日** | 週日（強制休） | 出勤即 1 倍 + 補休 8 小時，超 8 小時加倍 |
| **國定假日** | `holidays-client.isHoliday() && caption !== ''` | 前 8 小時等同 1 日工資，後續加班分段 |

> ⚠️ `getDayKind()` 既有實作回傳 `public/regular/rest/workday`，已對應上面 4 類

### 平日工時段

| 段別 | 範圍 | 倍率 |
|---|---|---|
| 正常工時 | 0 - 8 小時 | 1.0 |
| 加班 OT1 | 8 - 10 小時 | **1.34** (4/3) |
| 加班 OT2 | 10 小時 - | **1.67** (5/3) |

### 休息日工時段

| 段別 | 範圍 | 倍率 |
|---|---|---|
| REST_OT1 | 0 - 2 小時 | 1.34 |
| REST_OT2 | 2 - 8 小時 | 1.67 |
| REST_OT3 | 8 - 12 小時 | **2.67** (8/3) |

### 例假日

| 條件 | 計算 |
|---|---|
| 工時 1-8 小時 | 1 日工資 (8h × 時薪) + 補休折現 (8h × 時薪) = 16h |
| 工時 > 8 小時 | 1 日工資 + 補休 + 超時 × 2 倍 |

### 國定假日

| 條件 | 計算 |
|---|---|
| 工時 1-8 小時 | 1 日工資 (8h × 時薪) |
| 第 9-10 小時 | × 1.34 |
| 第 11 小時起 | × 1.67 |

### 淨工時扣除

- 總工時 = `punchOutTime - punchInTime`（同日內）
- 扣除：與 `breakTimes` 設定的休息時段重疊的分鐘數
- 淨工時 = 總工時 - 重疊休息分鐘

### 4. 員工設定：加「薪資制度」與「勞保投保等級」

**雙制度支援**

依勞基法施行細則第 31 條，月薪制員工的「平日每小時工資額」算法：

| 制度 | 輸入欄位 | 時薪換算 |
|---|---|---|
| 月薪制 | `monthlySalary` (NT$/月) | `hourlyRate = monthlySalary ÷ 30 ÷ 8`（=÷ 240）|
| 時薪制 | `hourlyRate` (NT$/小時) | 直接使用 |

**勞動部官方範例**
- 月薪 36,000 → 時薪 = 36,000 ÷ 240 = **150 元**
- 月薪 28,590（2026 年基本工資）→ 時薪 = 28,590 ÷ 240 ≈ **119 元**
- 法定基本時薪（2026）= **190 元**（時薪制下限）

加班費 = 時薪 × 加班時數 × 倍率（不論月薪/時薪都用換算後的時薪計）

**Firestore schema 擴充**

```js
// employees/{userId}
{
  // ...既有欄位
  salaryType: 'monthly' | 'hourly',  // 預設 'monthly'
  monthlySalary: 36000,              // monthly 模式必填
  hourlyRate: 150,                   // hourly 模式必填；monthly 模式由系統算填

  // === 勞保／健保／勞退（2026 年制度）===
  // 投保等級表內的「月投保薪資」級距，存等級代碼 + 對應金額
  laborInsuranceGrade: 17,           // 等級 1-23（依勞保投保薪資分級表）
  laborInsuranceSalary: 45800,       // 該等級對應的「月投保薪資」(NT$)
  // (健保等級多數同勞保，但有差別時可分開存；先共用)
  hasLaborPension: true,             // 是否提繳勞退（預設 true）
  laborPensionRate: 6,               // 提繳率 %（員工自願提繳 0-6，雇主固定 6）
}
```

**勞保投保薪資分級表**（2026 年版，台灣勞保局公告）

| 級 | 月投保薪資 | 適用月薪範圍 |
|---|---|---|
| 1 | 28,590 | ≤ 28,590（基本工資底線）|
| 2 | 30,300 | 28,591 – 30,300 |
| 3 | 31,800 | 30,301 – 31,800 |
| ... | ... | ... |
| 17 | 45,800 | 43,901 – 45,800 |
| ... | ... | ... |
| 23 | 72,800 | ≥ 72,800（最高級）|

> 完整 23 級表寫成常數 `LABOR_INSURANCE_GRADES` 放 `js/labor-hours.js`，
> 並提供 `inferGradeFromSalary(monthlySalary)` 自動推算。

**員工自付費率（依 2026 年費率，可由設定 override）**

| 項目 | 費率 | 員工負擔比例 | 員工每月扣 |
|---|---|---|---|
| 勞保普通事故保險費 | 12% | 20% | `投保薪資 × 12% × 20% = 2.4%` |
| 健保費 | 5.17% | 30% | `投保薪資 × 5.17% × 30% = 1.55%` |
| 勞退自提（自願） | 0–6% | 員工自選 | `投保薪資 × 自提率%` |

**雇主負擔（提供管理員看，員工不顯示）**
- 勞保 12% × 70% = 8.4%
- 健保 5.17% × 60% = 3.10%（眷屬視情況）
- 勞退 6%（強制）

## 風險與注意事項

1. **休息扣除的精確度**：punchInTime/punchOutTime 是 'HH:MM'（分鐘級），
   重疊時段算分鐘數即可，避免秒級複雜化
2. **跨日打卡**：dailyStatus.summarizeByDay 已分日，理論上每筆都同日；
   若 punchOut < punchIn 視為「下班 < 上班」+ 跳出計算（hours = 0）
3. **休息日國定假日重疊**：例如 2026 元旦在週日 → 國定假日優先
   （getDayKind 已優先 public）
4. **補班日**：API 的 `caption` 含「補班」字眼 → 視為平日（不適用國定假日規則）
5. **資料一致性**：dailyStatus.hours 是「淨工時」？需確認後端 summarizeByDay
   是否已扣休息。先假設「未扣」，由前端再扣，避免雙重扣除
6. **薪資是敏感資料**：時薪/月薪/勞保等級只在管理員 dashboard 顯示，
   getEmployeeList API 對非管理員 session 回傳時必須剝除 salary 欄位
7. **法定費率每年可能調整**：勞保 12% / 健保 5.17% / 基本工資 28,590 都
   寫常數 in `js/labor-hours.js`，註解年份；每年元旦前由開發者手動更新
8. **時薪 vs 月薪換算的爭議**：勞動部目前採「÷ 30 ÷ 8」公式，但近年
   有判決傾向「÷ 當月實際工作日 ÷ 8」更精準。本系統先採法規公式，
   未來如需精算可加 advance setting
9. **月薪未達基本工資**：UI 紅字警示且阻擋儲存，預設 placeholder 顯示
   28,590（2026 年基本月薪）
10. **眷屬健保 / 二代健保補充保費**：本計畫不含，留待「薪資結算系統」
