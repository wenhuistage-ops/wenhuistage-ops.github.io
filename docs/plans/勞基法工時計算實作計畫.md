# 勞基法工時計算實作計畫

> 建立：2026-04-26
> 狀態：規劃中
> 既有規則文件：[薪資與加班計算規則整理.md](../rules/薪資與加班計算規則整理.md)
> 既有 client 元件：`holidays-client.js` (`getDayKind`)、`weekly-chart.js`、`getBreakTimes`

---

## 目標

把目前管理員 dashboard 的「工時」單一數字（`day.hours`）細分為勞基法
規則下的多段時數，讓管理員一眼看到：

1. 每日：**正常 / 平日加班 1.34× / 平日加班 1.67× / 休息日 / 例假日 / 國定假日**
2. 全月 KPI：**正常時數、平日加班時數、休息日時數、國定假日時數、例假日時數**
3. 週圖：模式增加「平日加班 / 假日工時」
4. 表格：工時欄展開顯示分段
5. （未來）薪資估算：依時薪 × 倍率計算月薪

**完全不動後端** — 規則用既有 dailyStatus + holidays-client + breakTimes
在前端計算。後端薪資結算（月底報表）暫不在範圍。

## 規則對齊（摘自 docs/rules/薪資與加班計算規則整理.md）

### 日期類型分類（4 種）

| 類型 | 判斷 | 計算規則 |
|---|---|---|
| **平日** | 週一-五 + 非國定假日 + 非補班補休 | 8 小時內正常 / 加班分段 |
| **休息日** | 週六（無國定假日覆蓋） | 工時全為加班，3 段倍率 |
| **例假日** | 週日（強制休） | 出勤即 1 倍 + 補休 8 小時，超 8 小時加倍 |
| **國定假日** | `holidays-client.isHoliday() && caption !== ''` | 前 8 小時等同 1 日工資，後續加班分段 |

> ⚠️ `getDayKind()` 既有實作回傳 `public/regular/rest/workday`，已對應上面 4 類

### 平日工時段

| 段別 | 範圍 | 倍率 |
|---|---|---|
| 正常工時 | 0 - 8 小時 | 1.0 |
| 加班 OT1 | 8 - 10 小時 | **1.34** (4/3) |
| 加班 OT2 | 10 小時 - | **1.67** (5/3) |

### 休息日工時段

| 段別 | 範圍 | 倍率 |
|---|---|---|
| REST_OT1 | 0 - 2 小時 | 1.34 |
| REST_OT2 | 2 - 8 小時 | 1.67 |
| REST_OT3 | 8 - 12 小時 | **2.67** (8/3) |

### 例假日

| 條件 | 計算 |
|---|---|
| 工時 1-8 小時 | 1 日工資 (8h × 時薪) + 補休折現 (8h × 時薪) = 16h |
| 工時 > 8 小時 | 1 日工資 + 補休 + 超時 × 2 倍 |

### 國定假日

| 條件 | 計算 |
|---|---|
| 工時 1-8 小時 | 1 日工資 (8h × 時薪) |
| 第 9-10 小時 | × 1.34 |
| 第 11 小時起 | × 1.67 |

### 淨工時扣除

- 總工時 = `punchOutTime - punchInTime`（同日內）
- 扣除：與 `breakTimes` 設定的休息時段重疊的分鐘數
- 淨工時 = 總工時 - 重疊休息分鐘

## UI 變更

### 1. KPI 卡片：4 格 → 6 格 / 或下拉式詳細模式

選項 A：直接擴成 6 格（手機改 2-col）
```
| 平日工時 | 平日加班 | 休息日加班 | 國定假日 | 例假日 | 請假天數 |
```

選項 B：保留 4 格，在 KPI 下方加摺疊「工時詳細」展開上面 6 格
**推薦 B** — 桌機/手機都不擠

### 2. 打卡紀錄表格：「工時」欄展開

每一 row 「工時」欄改為：
```
4.0 (淨)
└─ 平日｜OT1 +0｜OT2 +0
```
桌機：直接展開 sub-row
手機：點開展開

### 3. 週工時圖：增加 mode

既有：`normal / overtime / rest / total`
新增：
- `平日加班` (ot1 + ot2 合計)
- `休息日` (含 ot1+ot2+ot3)
- `國定假日` (含保證 8h)
- `例假日` (含補休 16h 等)

### 4. 員工設定：加「時薪」欄位

```html
<input id="employee-hourly-rate" type="number" placeholder="時薪 (NT$)" />
```

存於 employees collection 的 `hourlyRate` 欄位。
有時薪後可顯示 KPI「本月應發薪資」。

### Phase L1：核心 lib（小，~1 小時）

- 寫 `js/labor-hours.js`：
  - `enrichDayWithLaborStats(day, breakTimes)` 計算單日 6 段
  - `aggregateMonthLaborStats(days)` 累計月度
- 單元邏輯測試（jest 或 console 驗證）

**驗收**：給林杰 4 月 25 天，能算出正常 X、平日加班 Y、休息日 Z、國定 N

---

### Phase L2：載入 breakTimes（小，~30 分鐘）

- admin.js 在 select 員工時 fetch 一次 `getBreakTimes`
- cache 在 module-level（公司設定不常變）
- 各 render function 共用

- admin.js loadMonthDetailData 之後，map 每筆呼叫 enrichDayWithLaborStats

- 後續所有 render 用 enriched 版本

---

### Phase L4：打卡紀錄表格 工時細項（中，~1 小時）

- 表格 row 加副行 `平日｜OT1｜OT2`

## 風險與注意事項

1. **休息扣除的精確度**：punchInTime/punchOutTime 是 'HH:MM'（分鐘級），
   重疊時段算分鐘數即可，避免秒級複雜化
2. **跨日打卡**：dailyStatus.summarizeByDay 已分日，理論上每筆都同日；
   若 punchOut < punchIn 視為「下班 < 上班」+ 跳出計算（hours = 0）
3. **休息日國定假日重疊**：例如 2026 元旦在週日 → 國定假日優先
   （getDayKind 已優先 public）
4. **補班日**：API 的 `caption` 含「補班」字眼 → 視為平日（不適用國定假日規則）
5. **資料一致性**：dailyStatus.hours 是「淨工時」？需確認後端 summarizeByDay
   是否已扣休息。先假設「未扣」，由前端再扣，避免雙重扣除
6. **薪資是敏感資料**：時薪只在管理員 dashboard 顯示，不傳前端 cache 給其他員工

## 不在本計畫範圍

- 月底結薪報表（Excel 匯出）
- 銀行匯款檔產生
- 個人/眷屬扣繳設定
- 加班費上報勞動局
- 工資清冊保存

這些屬於「薪資結算系統」獨立模組，留待後續迭代。
