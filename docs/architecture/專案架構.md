# 專案架構分析

## 📋 項目概述
**0riginAttendance-System** 是一個開源的考勤管理系統，基於 LINE Login 和 Google Apps Script（GAS）。用戶通過 LINE 帳號登入，使用手機 GPS 進行打卡，打卡記錄實時儲存在 Google 試算表中。

- **專案名稱**：attendance-system
- **版本**：1.0.0
- **授權**：GPL v2
- **作者**：0J (Lin Jie / 0rigin1856)

## 📌 核心功能模塊

### **1. 認證系統（Authentication）**
- **技術棧**：LINE Login OAuth 2.0
- **流程**：
  1. 前端呼叫 `getLoginUrl()` 取得 LINE 授權 URL
  2. 用戶在 LINE 認證後，後端獲取授權碼
  3. 後端調用 LINE API 獲取用戶資訊，生成 Session Token
  4. 前端存儲 Token 到 `localStorage`
  5. 後續 API 調用都攜帶 Token 進行驗證

### **2. 定位打卡系統（GPS Punch Clock）**
- **主要文件**：`js/punch.js`、`js/location.js`
- **核心功能**：
  - 使用 `navigator.geolocation` 獲取用戶 GPS 坐標（支援高精確度模式）
  - 實作精確度檢查和自動重試機制（最多3次重試）
  - 驗證打卡點是否在允許範圍內（基於打卡地點座標＋半徑）
  - 記錄上班、下班、補卡時間到 Google Sheets

### **3. 管理員面板（Admin Panel）**
- **文件**：`js/admin.js`、HTML 中的 admin tab
- **功能**：
  - 新增、編輯、管理打卡地點（經緯度、半徑）
  - 審核、駁回、批准員工補卡申請
  - **⚠️ 安全隱患**：目前對所有用戶開放，未做權限檢查

### **4. 多國語言系統（i18n）**
- **技術**：JSON 翻譯文件 + `t()` 函數
- **支持語言**：繁體中文、日文、英文、越南文
- **實現方式**：
  - 靜態內容使用 `data-i18n` 標籤
  - 動態內容使用 `data-i18n-key` 與 JavaScript 綁定
  - 自動根據瀏覽器語言選擇

### **5. 異常檢查系統（Anomaly Detection）**
- **功能**：自動檢查打卡記錄異常（如同日期重複打卡、時間邏輯問題）
- **實現**：後端驗證或前端提示

## 🔄 請求流程圖

### **打卡流程**
```
1. 用戶點擊 "上班打卡"
   ↓
2. js/punch.js 調用 navigator.geolocation 獲取位置
   ↓
3. 驗證位置是否在允許範圍內
   ↓
4. 若有效，調用 API：callApifetch({ action: 'punch', ... })
   ↓
5. GAS 的 Main.gs → doPunch() → DbOperations.gs 寫入 Google Sheets
   ↓
6. 返回成功/失敗訊息，前端顯示通知
```

### **認證流程**
```
1. 前端檢查 localStorage 是否有 sessionToken
   ↓
2. 若無，呼叫 getLoginUrl()，重定向到 LINE 認證頁
   ↓
3. 用戶授權後，LINE 重定向回應用 + 授權碼
   ↓
4. GAS 處理授權碼，與 LINE API 交互
   ↓
5. 生成 sessionToken 存入 Google Sheets
   ↓
6. 前端往後請求時都攜帶 sessionToken
```

## 📊 API 端點清單

| 端點 | 方法 | 功能 |
|------|------|------|
| `/exec?action=getLoginUrl` | GET | 獲取 LINE 認證 URL |
| `/exec?action=getProfile&code=XXX` | GET | 獲取用戶資訊 |
| `/exec?action=checkSession&token=XXX` | GET | 驗證 Session 令牌 |
| `/exec?action=punch` | POST | 執行打卡 |
| `/exec?action=getPunchRecords` | GET | 查詢打卡記錄 |
| `/exec?action=getLocations` | GET | 查詢打卡地點 |
| `/exec?action=addLocation` | POST | 新增打卡地點 |
| `/exec?action=checkAbnormal` | GET | 檢查異常記錄 |
| `/exec?action=submitMakePunch` | POST | 提交補卡申請 |

## 🔐 安全與權限機制

### **現有機制**
- Session Token 驗證（TTL-based）
- localStorage 中儲存 `isAdmin` 狀態
- 部分 API 檢查用戶身份

### **已知問題**
- ⚠️ 管理員面板對所有用戶開放（無前端檢查）
- localStorage 中的 `isAdmin` 可被本地修改
- 應在後端嚴格驗證所有敏感操作
