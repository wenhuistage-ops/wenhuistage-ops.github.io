## 三、健康度評估

### ✅ 既有優勢
1. **建置現代化** — Vite + Tailwind 4 + Jest 30，工具鏈未過時
2. **多語完備** — 5 種語系支援，i18n.js 已模組化
3. **快取策略** — cache.js 已分層設計，避免不必要的 API 呼叫

5. **文件豐富** — 13 份 .md 涵蓋規則、架構、安全、效能

## 五、優化路徑（分階段）

### 🥇 階段一：清理與奠基（1–3 天，零風險）

| # | 工作項 | 預期效益 |
|---|-------|----------|
| 1 | 執行「4.1 立即可執行」三個動作 | 倉庫整潔，新人易讀 |
| 2 | 執行「4.2 docs 目錄重組」 | 文件可維護性提升 |
| 3 | 統一 state.js（保留 modules/state.js，淘汰 js/state.js） | 消除雙源 |
| 4 | 補上 docs/ChangeLog.md | 變更可追溯 |

### 🥈 階段二：模組拆分（1–2 週，中風險）

#### admin.js（2,511 行）拆分藍圖

```
js/admin.js  →  js/admin/
              ├── index.js              # 入口、事件繫結（<200 行）
              ├── salary-calculator.js  # 薪資/加班計算（已部分在 modules/payroll.js）
              ├── overtime-rules.js     # 加班倍率規則
              ├── approval-panel.js     # 管理員審核 UI
              ├── export-handler.js     # XLSX 匯出（與 punch 共用）
              └── filter-controller.js  # 條件查詢
```

#### punch.js（1,029 行）拆分藍圖

```
js/punch.js  →  js/punch/
              ├── index.js          # 對外公開 API（<150 行）
              ├── gps-validator.js  # GPS 距離與精度驗證
              ├── punch-flow.js     # 上下班/加班打卡流程
              ├── make-up.js        # 補卡流程
              └── notifier.js       # 與通知佇列整合
```

#### index.html（758 行）模板抽出

```
index.html  →  templates/
              ├── login.html
              ├── punch-card.html
              ├── admin-panel.html
              └── modal-dialogs.html
（透過 template-loader.js 動態載入）
```

### 🥉 階段三：效能轉型（2–4 週，高效益）

依據既有 `Firestore遷移計劃.md`：

| 階段 | 目標 |
|------|------|
| 3-1 | 通知系統異步化（依 `異步通知系統.md`），目標 5.8s → 3s |
| 3-2 | 讀取路徑遷移至 Firestore，目標 3s → 800ms |
| 3-3 | 寫入路徑雙寫驗證 |
| 3-4 | GAS 退役為純通知/排程，移除冷啟動瓶頸 |

### 🏅 階段四：品質回補（持續）

1. **重啟 Jest 覆蓋率閥值** — 模組化後恢復 `coverageThreshold`，初始建議 50%
2. **為核心計算寫測試** — salary-calculator、overtime-rules、gps-validator 為首要對象
3. **加上 CI** — GitHub Actions：lint + test + build
4. **效能監控** — 在 `core.js` API 封裝層埋點，輸出至 console.timing
