## 📋 當前問題分析

### 問題 1：「請假(審核中)」顯示「補打卡」按鈕

**現象**：
- 日期：2026-04-07, 2026-04-04, 2026-04-02
- 狀態：紅色標籤「請假(審核中)」+ 藍色「補打卡」按鈕
- **期望**：不應該顯示「補打卡」，因為已有請假申請待審核

**業務含義**：
- 當用戶提交了「請假」或「休假」申請時，即使還在審核中
- 該日期應該被認為「已有說明」，不需要補打卡
- 用戶應該等待批准/拒絕，而非再提交補打卡

### 問題 2：「已批准的請假」仍在異常清單中

**現象**：
- 日期：2026-04-05
- 狀態：紅色標籤「本日未打卡」+ 藍色「補打卡」按鈕  
- **期望**：不應該在異常清單中，因為請假已批准

**業務含義**：
- 當請假/休假被管理員批准後
- 該日期應該被視為「正常」（已付薪休息日）
- 不應該再出現在異常清單中

### 2. 按鈕顯示規則

#### 補打卡按鈕
**顯示條件**（三個都須滿足）：
```
1. 異常原因為 STATUS_BOTH_MISSING, STATUS_PUNCH_IN_MISSING, 或 STATUS_PUNCH_OUT_MISSING
2. 沒有待審核的補卡申請 (STATUS_REPAIR_PENDING)
3. 沒有待審核或已批准的請假/休假申請
```

**不顯示條件**（任一即隱藏）：
- 異常原因為 `STATUS_LEAVE_PENDING` 或 `STATUS_VACATION_PENDING`（已有請假申請）
- 異常原因為 `STATUS_REPAIR_PENDING`（補卡已申請）
- 該日期有待審核的申請 (status='pending' 或 'reviewing')

#### 請假/休假按鈕
**顯示條件**（三個都須滿足）：
```
1. 異常原因為 STATUS_BOTH_MISSING（完全未打卡）
2. 沒有待審核的補卡申請
3. 沒有待審核或已批准的請假/休假申請
```

**不顯示條件**（任一即隱藏）：
- 該日期只缺一張卡（上班卡或下班卡）
- 該日期有待審核的申請
- 該日期已有請假/休假申請（無論是否批准）

## 🔧 後端邏輯缺陷

### 缺陷 1：未區分請假的批准狀態

**當前實現**（Utils.gs, 行 173-174）:
```javascript
const isAllApproved = (totalAdjustments > 0 && approvedAdjustmentCount === totalAdjustments) || 
                     (totalLeaveRequests > 0 && approvedLeaveCount === totalLeaveRequests);
```

**問題**：
- 當有請假記錄且都被批准時，異常原因設為 `STATUS_REPAIR_APPROVED`
- 但實際上應該進一步區分：
  - `STATUS_LEAVE_APPROVED`（請假已批准）
  - `STATUS_VACATION_APPROVED`（休假已批准）
- 這樣前端才能安全地排除它們

**應改為**：
```javascript
// 首先檢查是否有批准的請假/休假
const hasApprovedLeave = hasLeaveRequest && approvedLeaveCount === totalLeaveRequests;
const hasApprovedVacation = hasLeaveRequest && approvedLeaveCount === totalLeaveRequests && isVacation;

// 其次檢查是否有批准的補卡
const hasApprovedRepair = totalAdjustments > 0 && approvedAdjustmentCount === totalAdjustments;

if (hasApprovedLeave) {
    reason = "STATUS_LEAVE_APPROVED";
} else if (hasApprovedVacation) {
    reason = "STATUS_VACATION_APPROVED";
} else if (hasApprovedRepair) {
    reason = "STATUS_REPAIR_APPROVED";
}
```

### 缺陷 2：異常清單仍包含已批准的申請

**當前實現**（Utils.gs, 行 206-214）:
```javascript
if (reason && reason !== "STATUS_PUNCH_NORMAL" && reason !== "STATUS_REPAIR_APPROVED") {
    // 只添加如果不是規則狀態
    abnormalRecords.push({...});
}
```

**問題**：
- 只排除了 `STATUS_PUNCH_NORMAL` 和 `STATUS_REPAIR_APPROVED`
- 但沒有排除新增的 `STATUS_LEAVE_APPROVED` 和 `STATUS_VACATION_APPROVED`

**應改為**：
```javascript
const normalStatuses = [
    "STATUS_PUNCH_NORMAL",
    "STATUS_REPAIR_APPROVED",
    "STATUS_LEAVE_APPROVED",
    "STATUS_VACATION_APPROVED"
];

if (reason && !normalStatuses.includes(reason)) {
    abnormalRecords.push({...});
}
```

## 🎨 前端邏輯缺陷

### 缺陷 1：補打卡按鈕未考慮請假狀態

**當前實現**（js/punch.js, 行 495, 507-512）:
```javascript
const showLeaveButtons = record.reason === "STATUS_BOTH_MISSING";

if (!hasPendingApplication) {
    buttonsHtml = `<button class="adjust-btn">補打卡</button>`;
}
```

**問題**：
- 只檢查 `hasPendingApplication`（來自 record.status）
- 未檢查 `record.reason` 是否為 `STATUS_LEAVE_PENDING` 或 `STATUS_VACATION_PENDING`
- 導致「請假審核中」的日期仍顯示補打卡按鈕

**應改為**：
```javascript
const showAdjustBtn = !hasPendingApplication && 
    !["STATUS_LEAVE_PENDING", "STATUS_VACATION_PENDING"].includes(record.reason);

if (showAdjustBtn) {
    buttonsHtml = `<button class="adjust-btn">補打卡</button>`;
}
```

### 缺陷 2：請假/休假按鈕未檢查已有申請

**當前實現**（js/punch.js, 行 515）:
```javascript
if (showLeaveButtons && !hasPendingApplication) {
    // 顯示請假和休假按鈕
}
```

**問題**：
- `showLeaveButtons` 只檢查是否為 `STATUS_BOTH_MISSING`
- 未檢查是否已有 `STATUS_LEAVE_PENDING` 或 `STATUS_VACATION_PENDING` 的申請
- 未檢查是否已批准（應從異常清單中消失）
