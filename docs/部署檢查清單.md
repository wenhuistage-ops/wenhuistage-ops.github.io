## 2. Firestore 資料準備

### 2.1 員工資料補充欄位（新功能依賴）

對 `employees/{userId}` 建議手動補上預設值（不補也能跑，但會顯示「未設定」）：

| 欄位 | 類型 | 預設 | 用途 |
|---|---|---|---|
| `salaryType` | string | 'monthly' | 'monthly' / 'hourly' |
| `monthlySalary` | number | 0 | 月薪 NT$ (≥ 28590 = 2026 基本工資) |
| `hourlyRate` | number | 0 | 時薪 NT$/h（hourly 模式或自動換算）|
| `laborInsuranceGrade` | number(1-23) | null | 勞保投保等級 |
| `hasLaborPension` | boolean | true | 是否提繳勞退 |
| `laborPensionRate` | number(0-6) | 0 | 員工自提率 % |
| `dept` | string | '一般員工' | '管理員' / '一般員工' |
| `status` | string | '啟用' | '啟用' / '停用' |

> 推薦做法：先讓管理員透過新建的「員工設定 → 薪資與勞保」UI 一個個設定，UI 會自動推算等級。

### 2.2 公司休息時段（已存在不必動）

`settings/breakTimes` 文件，schema：
```
{ breaks: [{ name, start, end }, ...] }
```
若要更動可從「打卡管理 → 休息時間設定」UI 操作。
