# 問題分析與建議 (2026年4月更新)

## 🚨 已發現的重要問題

### **1. 權限與安全隱患 [🔴 高優先級]**

#### **問題 1.1：管理員面板無身份驗證**
- **狀態**：❌ **未解決** - 仍需改進
- **描述**：Admin Tab 依然只依賴 localStorage 中的 isAdmin 狀態，未每次驗證真實權限
- **位置**：`js/app.js` 第 209-217 行
- **風險**：用戶可在開發者工具修改 localStorage，偽裝成管理員
- **當前代碼**：
```javascript
tabAdminBtn.addEventListener('click', () => {
    const isUserAdmin = (localStorage.getItem("isAdmin") === 'true');
    if (isUserAdmin) {
        switchTab('admin-view');
    } else {
        showNotification(t("ERR_NO_PERMISSION"), "error");
    }
});
// ❌ 只檢查本地 localStorage，未驗證服務器
```

**改善方案**：
```javascript
// ✅ 每次點擊都驗證真實權限
tabAdminBtn.addEventListener('click', async () => {
    const res = await callApifetch({ action: 'checkAdminStatus' });
    if (!res.isAdmin) {
        showNotification(t("ERR_NO_PERMISSION"), "error");
        return;
    }
    switchTab('admin-view');
});
```

**修復優先度**：🔴 **立即** | **預計工時**：1 小時 | **責任人**：前端

---

#### **問題 1.2：localStorage 中存儲 isAdmin 狀態**
- **狀態**：❌ **未解決** - 仍存在安全隱患
- **描述**：`isAdmin` 狀態存在 localStorage 並被多處依賴，容易被修改
- **位置**：`js/app.js` 第 43、50、184、300 行，被多個地方使用
- **風險**：用戶可手動修改為 `true`，繞過權限檢查
- **建議**：改為每次需要時從服務器查詢

**改善方案**：
```javascript
// ✅ 不存儲權限狀態，每次查詢時驗證
async function isCurrentUserAdmin() {
    const res = await callApifetch({ action: 'checkAdminStatus' });
    return res.isAdmin;
}

// 使用時
if (await isCurrentUserAdmin()) {
    // 執行管理員操作
}
```

**修復優先度**：🔴 **立即** | **預計工時**：2 小時 | **責任人**：前端

#### **問題 4.1：全域變數污染（state.js）**
- **狀態**：❌ **未改進** - 仍有大量全域變數
- **描述**：大量全域變數定義在 `js/state.js` 中，難以維護
- **位置**：`js/state.js` 及各 JS 文件
- **影響**：代碼混亂，容易產生命名衝突，難以追蹤狀態變化
- **當前狀況**：仍然使用全域變數模式

**改善方案**：
```javascript
// ✅ 改為 Class 模式
class AppState {
  constructor() {
    this.ui = {};
    this.user = {};
  }
}
const appState = new AppState();
```

**修復優先度**：🟡 **下月** | **預計工時**：4-6 小時 | **責任人**：前端重構

---

#### **問題 4.2：缺少錯誤邊界（Error Boundary）**
- **狀態**：❌ **未實現**
- **描述**：API 調用缺少統一的錯誤處理
- **位置**：各 `.js` 文件中的 `callApifetch()` 調用
- **風險**：未捕獲的錯誤導致應用崩潰

**改善方案**：
```javascript
// ✅ 統一錯誤處理包裝器
async function callApiWithErrorHandling(params) {
  try {
    const response = await callApifetch(params);
    if (!response.ok) {
      showNotification(t(response.code || "API_ERROR"), "error");
      return null;
    }
    return response;
  } catch (err) {
    showNotification(t("NETWORK_ERROR"), "error");
    return null;
  }
}
```

**修復優先度**：🟡 **下月** | **預計工時**：2-3 小時 | **責任人**：前端
