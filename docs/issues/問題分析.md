# 問題分析與建議

## 🚨 已發現的重要問題

### **1. 權限與安全隱患 [高優先級]**

#### **問題 1.1：管理員面板無身份驗證**
- **描述**：Admin Tab 對所有用戶開放，未檢查是否真正為管理員
- **位置**：`js/admin.js` 中的 Admin Tab 初始化
- **風險**：任何用戶都可以訪問管理員功能（新增地點、審核補卡等）
- **影響**：資料完整性、機密性受損

**當前代碼檢查**：
```javascript
// 在 app.js 中
if (isAdmin) {
    document.getElementById('tab-admin-btn').style.display = 'block';
}
// 但 isAdmin 是從 localStorage 讀取，可被本地修改！
```

**改善方案**：
```javascript
// ✅ 每次打開管理員面板時，都應驗證
async function verifyAdminAccess() {
    const res = await callApifetch({ action: 'checkAdminStatus' });
    if (!res.isAdmin) {
        showNotification("❌ 無管理員權限", "error");
        return false;
    }
    return true;
}

// 在管理員操作前調用
document.getElementById('admin-actions').addEventListener('click', async (e) => {
    if (!(await verifyAdminAccess())) return;
    // 執行管理員操作
});
```

#### **問題 2.2：Session TTL 管理不當**
- **描述**：Session 過期後未能完全清理，可能導致"殭屍" Session
- **位置**：`GS/DbOperations.gs` 中的 `checkSession_()` 函數
- **風險**：舊 Session 記錄堆積，資料庫膨脹

**改善方案**：
```javascript
// 每次 checkSession 時，同步清理過期記錄
function checkSession_(sessionToken) {
  const sheet = SpreadsheetApp.getActive().getSheetByName(SHEET_SESSION);
  const values = sheet.getDataRange().getValues();
  
  const now = new Date();
  const rowsToDelete = [];
  
  // ✅ 清理過期 Session
  for (let i = values.length - 1; i >= 1; i--) {
    if (new Date(values[i][3]) < now) {
      rowsToDelete.push(i);
    }
  }
  
  // 倒序刪除以保持索引準確
  rowsToDelete.forEach(i => sheet.deleteRow(i));
}
```

### **3. GPS 定位驗證問題 [中優先級]**

#### **問題 3.1：缺少定位失敗的降級方案**
- **描述**：用戶拒絕共享位置或 GPS 信號不佳時，應用無法打卡
- **位置**：`js/location.js` 中的 `getLocation()` 函數
- **風險**：用戶無法打卡，實際上班卻被記為缺卡

**改善方案**：
```javascript
async function getLocationWithFallback() {
    try {
        return await new Promise((resolve, reject) => {
            navigator.geolocation.getCurrentPosition(
                (position) => resolve(position.coords),
                (error) => {
                    if (error.code === error.PERMISSION_DENIED) {
                        // ✅ 提示用戶需要啟用位置權限
                        showNotification(t("LOCATION_PERMISSION_DENIED"), "error");
                    } else if (error.code === error.TIMEOUT) {
                        // ✅ GPS 超時，提示重試
                        showNotification(t("LOCATION_TIMEOUT"), "warning");
                    }
                    reject(error);
                },
                { timeout: 10000, maximumAge: 60000 }
            );
        });
    } catch (err) {
        // ✅ 允許管理員手動授權（無 GPS 打卡）
        const proceed = confirm(t("PROCEED_WITHOUT_GPS"));
        if (proceed) {
            return { latitude: 0, longitude: 0, accuracy: -1 }; // 標記為無定位打卡
        }
        throw err;
    }
}
```

#### **問題 3.2：打卡距離驗證邏輯缺失**
- **描述**：未查看實際的距離計算函數，可能存在計算誤差
- **位置**：`GS/Utils.gs` 中的距離計算
- **建議**：驗證使用 Haversine 公式（球面距離計算）

**示例實現**：
```javascript
function calculateDistance_(lat1, lon1, lat2, lon2) {
  const R = 6371000; // 地球半徑（米）
  const dLat = toRad(lat2 - lat1);
  const dLon = toRad(lon2 - lon1);
  const a = 
    Math.sin(dLat / 2) * Math.sin(dLat / 2) +
    Math.cos(toRad(lat1)) * Math.cos(toRad(lat2)) * 
    Math.sin(dLon / 2) * Math.sin(dLon / 2);
  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
  return R * c; // 米
}

function toRad(deg) {
  return deg * (Math.PI / 180);
}
```

#### **問題 4.2：缺少錯誤邊界（Error Boundary）**
- **描述**：前端 API 調用缺少統一的錯誤處理
- **位置**：各 `.js` 文件中的 `callApifetch()` 調用
- **風險**：未捕獲的錯誤導致應用崩潰

**改善方案**：
```javascript
// ✅ 封裝 API 調用並添加統一錯誤處理
async function callApiWithErrorHandling(params) {
  try {
    const response = await callApifetch(params);
    
    if (!response.ok) {
      const errorMsg = t(response.code || "API_ERROR");
      showNotification(`❌ ${errorMsg}`, "error");
      console.error("API 錯誤:", response);
      return null;
    }
    
    return response;
  } catch (err) {
    console.error("API 請求失敗:", err);
    showNotification(t("NETWORK_ERROR"), "error");
    return null;
  }
}

// 使用示例
const profile = await callApiWithErrorHandling({ action: 'getProfile' });
if (!profile) return; // 靜默失敗，不執行後續邏輯
```

### **5. 後端架構問題 [中優先級]**

#### **問題 5.1：缺少輸入驗證**
- **描述**：GAS 後端未驗證前端傳來的參數
- **位置**：`GS/Main.gs` 中的各 API 處理函數
- **風險**：注入攻擊、SQL 語句注入（縱然用不到 SQL）

**改善方案**：
```javascript
// ✅ 在 Main.gs 中添加驗證層
function validatePunchRequest(e) {
  const action = e.parameter.action;
  const token = e.parameter.token;
  const lat = parseFloat(e.parameter.lat);
  const lng = parseFloat(e.parameter.lng);
  
  // 驗證必須參數
  if (!action || !token) {
    return { ok: false, code: "MISSING_PARAMS" };
  }
  
  // 驗證數據類型
  if (isNaN(lat) || isNaN(lng)) {
    return { ok: false, code: "INVALID_COORDINATES" };
  }
  
  // 驗證座標範圍（緯度 -90 ~ 90，經度 -180 ~ 180）
  if (lat < -90 || lat > 90 || lng < -180 || lng > 180) {
    return { ok: false, code: "OUT_OF_RANGE_COORDINATES" };
  }
  
  return { ok: true };
}

// 在 doPunch 中調用
function doPunch(e) {
  const validation = validatePunchRequest(e);
  if (!validation.ok) {
    return ContentService.createTextOutput(JSON.stringify(validation))
      .setMimeType(ContentService.MimeType.JSON);
  }
  
  // ... 後續邏輯 ...
}
```

#### **問題 5.2：API 返回格式不統一**
- **描述**：部分 API 返回 JSON，部分返回 JSONP，未統一
- **位置**：`GS/Main.gs` 中的 `respond()` 和 `respond1()` 函數
- **風險**：前端處理邏輯複雜且易出錯

**改善方案**：
```javascript
// GS/Main.gs 中統一 API 響應格式
function createResponse(ok, data = {}, code = "OK") {
  return {
    ok: ok,
    code: code,
    data: data,
    timestamp: new Date().toISOString()
  };
}

function respond(obj, callback) {
  const jsonp = `${callback}(${JSON.stringify(obj)})`;
  return ContentService.createTextOutput(jsonp)
    .setMimeType(ContentService.MimeType.JAVASCRIPT);
}

// 使用
function doPunch(e) {
  try {
    // ... 業務邏輯 ...
    return respond(createResponse(true, { punchTime: now }, "PUNCH_SUCCESS"), e.parameter.callback);
  } catch (err) {
    return respond(createResponse(false, {}, "INTERNAL_ERROR"), e.parameter.callback);
  }
}
```

### **6. 多語言系統問題 [低優先級]**

#### **問題 6.1：缺少翻譯完整性檢查**
- **描述**：某些語言文件可能缺少某些鍵值，導致顯示英文
- **位置**：`i18n/*.json` 文件
- **風險**：用戶體驗下降

**建議**：
```javascript
// 在 core.js 中添加檢查
async function loadTranslations(lang) {
    const res = await fetch(`https://wenhuistage-ops.github.io/i18n/${lang}.json`);
    if (!res.ok) {
        // ✅ 降級到預設語言（英文）
        loadTranslations('en-US');
        return;
    }
    translations = await res.json();
    
    // ✅ 檢查缺少的鍵值
    const missingKeys = checkMissingKeys(translations);
    if (missingKeys.length > 0) {
        console.warn(`語言 ${lang} 缺少鍵值:`, missingKeys);
    }
}

function checkMissingKeys(translations) {
  // 定義所有必須的鍵值
  const requiredKeys = [
    "APP_TITLE", "LOGGING_IN", "PUNCH_IN", "PUNCH_OUT", 
    "INVALID_LOCATION", "LOCATION_PERMISSION_DENIED", // ... 等等
  ];
  
  return requiredKeys.filter(key => !(key in translations));
}
```

- **位置**：`GS/DbOperations.gs` 中的 `writePunchRecord_()`

**改善方案**：
```javascript
function writePunchRecord_(userId, date, punchType, location, coordinates) {
  const sheet = SpreadsheetApp.getActive().getSheetByName(SHEET_PUNCH_RECORDS);
  const values = sheet.getDataRange().getValues();
  
  // ✅ 檢查是否已存在同日期、同類型的記錄
  for (let i = 1; i < values.length; i++) {
    const existingDate = new Date(values[i][2]);
    const existingType = values[i][5]; // punchType 位於第 5 欄
    
    if (
      values[i][1] === userId &&
      existingDate.toDateString() === date.toDateString() &&
      existingType === punchType &&
      new Date() - new Date(values[i][12]) < 60000 // 60 秒內
    ) {
      return { ok: false, code: "DUPLICATE_PUNCH_DETECTED" };
    }
  }
```
