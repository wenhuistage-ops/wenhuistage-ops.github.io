# 問題分析與待辦清單

> 最後更新：2026-04-27
> 範圍：僅列**尚未解決**的問題；已完成項目請見 `docs/ChangeLog.md` 與 git log。

---

## 🟠 中優先級

### 2.1 sessionToken 存在 localStorage（XSS 風險）

- **背景**：2026-04-14 已實施方案 B（改用 POST，token 不再經 URL 傳遞）。
- **剩餘風險**：token 仍寫入 `localStorage`，任何 XSS 漏洞都能被竊取。
- **改善方向**（任選）：
  - 改存 IndexedDB + 短 TTL + 重新登入機制
  - 後端改發 httpOnly cookie（須調整 CORS / CSRF 設計）
- **位置**：[js/app.js](../../js/app.js)、[js/firestore-client.js](../../js/firestore-client.js)、`firebase-functions/functions/src/*`
- **預估工時**：4–6h

---

### 2.2 Session TTL 清理機制缺失

- **現況**：[`_helpers.js`](../../firebase-functions/functions/src/_helpers.js) 設 `SESSION_TTL_MS = 30 天`、寫入時帶 `expiredAt`，但**沒有清理過期文件的排程**。
- **影響**：`sessions` collection 持續成長，舊 token 雖然查驗會被 `verifySession` 擋掉但 doc 仍留著。
- **改善方向**：
  - 用 Cloud Functions scheduler 每天掃 `expiredAt < now` 批次刪除
  - 或改用 Firestore TTL policy（GCP 原生功能，零成本）
- **預估工時**：1–2h

---

## 🟡 低優先級（技術債）

### 4.1 全域變數污染（state.js）

- **現況**：[js/state.js](../../js/state.js) 仍以全域 `var/let` 暴露大量狀態（`userId`, `currentMonthDate`, `currentLanguage`...）。
- **影響**：難以追蹤狀態變化、模組之間隱性耦合。
- **改善方向**：包成 `AppState` class 或 `Object.freeze` 命名空間。
- **預估工時**：4–6h（接觸面廣）

---

### 4.2 缺少統一 API 錯誤邊界

- **現況**：各模組直接 `await callApifetch(...)`，錯誤處理散落各處。
- **改善方向**：包一層 `callApiWithErrorHandling`，統一 toast/log/重試。
- **預估工時**：2–3h

---

### 6.1 翻譯完整性檢查

- **現況**：[js/modules/i18n.js](../../js/modules/i18n.js) 載入後不會比對其他語言缺哪些鍵。
- **影響**：新增翻譯鍵時容易漏改其他 4 國語系，UI 直接顯示 raw key。
- **改善方向**：CI / 啟動時 diff 各語系 key set，缺漏時 console.warn。
- **預估工時**：1–2h

---

### 8.2 類型安全（JSDoc / TypeScript）

- **現況**：純 JS，無型別註解；Cloud Functions 端也沒有 JSDoc。
- **改善方向**：先補關鍵模組（labor-hours、firestore-client、_helpers）的 JSDoc，未來視情況評估 TS。
- **預估工時**：分批進行

---

### 8.5 結構化日誌（Logger）

- **現況**：全前端用 `console.log/warn/error`，無等級控制、無遠端收集。
- **改善方向**：簡單 Logger 封裝（debug/info/warn/error + production 時降低詳度）。
- **預估工時**：2h

---

### 9.1 API URL 硬編碼

- **現況**：[js/config.js](../../js/config.js) 寫死正式環境 GAS URL。
- **改善方向**：依 hostname 切換 dev/prod，或用 build-time 注入。
- **預估工時**：1h
